# v2-剧情模块0213

## 一、简介

剧情模块包含omao-v1的主页、剧情体验和回忆三个与live2d相关的内容。

- 剧情主页：原始omao-v1版本的主页Live2d模型交互（动画、语音、字幕等）相关的内容，移动到剧情主页（从主页的剧情入口进入剧情模块的页面），包含功能：
  - 隐藏按键
  - 剧情入口
  - 回忆页入口
  - 切换角色
  - 背景音乐
- v1版本剧情体验
  - 剧情选择页/剧情导航页
  - 剧情内页（live2d动画）
- 回忆页
  - 立绘展示
  - 文案展示
  - 语音展示

## 二、需求描述

### 1. 剧情主页

#### 1.1 剧情主页live2d交互

剧情主页背景为Live2D动画，根据实际不同Live2D模型，点击不同区域会做出对应的交互响应动画。点击角色播放对应的看板语音，同时显示对应的字幕，具体内容可见Live2d/语音交互效果说明。

live2d模型分为待机和主动交互两种状态，用户可以通过点击触发区域进行主动互动，在用户未操作的情况下，以固定时间间隔播放待机语音。

##### 1.1.1 待机

- 触发节点：刚进入剧情主页、剧情主页停留一段时间后、从剧情子页面切换回剧情主页后随机播放
- 动画内容：循环播放相应idle动画
- 语音：播放语音及字幕
  - 特定情形下播放待机语音
  - 不同权限的账号播放的语音内容不相同

##### 1.1.2 主动交互

以下为存在的功能说明，实际效果见Live2d/语音交互效果说明。主动交互播放完毕后，若没有其他触发，则恢复待机循环播放。

- 触发节点：用户点击交互区块
- 触发效果
  - 触发动画
    - 当点击区域有指定特定动画时，用户点击区域触发响应的动画
    - 当点击区域没有指定特定动画时，用户点击区域保持原始idle动画
  - 触发语音
    - 当点击区域有指定特定语音时，用户点击区域触发对应的语音和字幕
  - 触发的语音和动画可能对应多个，根据概率进行播放
- 剧情主页动画冲突：在主动播放语音和动画时，需要等待语音时间
  - 若当前有非idle动画，需要等待动画播放完毕，才能响应接下来的点击事件
  - 如果当前为idle动画，或者交互动画已经播放完毕，则停止当前语音，响应下一个点击事件

### 2. 剧情体验（命运航线）

#### 2.1 剧情选择页

剧情选择页面可以点击选择不同剧情。

- 当前账号有权限且已解锁的剧情卡片为指定图片
- 当前账号无权限/未解锁的剧情卡片为指定图片加锁

#### 2.2 剧情内页

##### 2.2.1 权限解锁

- 权限：进入具体的剧情内容体验权限
- 权限码解锁
  - 解锁前置条件：账号拥有需要解锁的剧情体验权限（用户获取剧情权限可见v2-主页及其他页面和页面权限）
  - 解锁节点
    - 第一段（1-1）：账号获取剧情体验权限时，解锁1-1剧情
    - 其他段：账号完成上一段剧情体验时，解锁下一段剧情
      - 1-1完成->解锁1-2；1-2完成->解锁1-3；...；1-5完成->解锁1-6
  - 如果用户失去对应的剧情权限，则该剧情变回锁定，不影响其他已解锁的内容
  - 举例
    - 用户通过指定方式获取所有的剧情权限后，首先已解锁1-1，随后用户完成剧情1-1后可解锁1-2...
    - 用户当前已经解锁1-1,1-2,1-3,1-4，此时用户失去1-2权限，则1-2剧情变为锁定，1-3,1-4仍然为已解锁剧情

##### 2.2.2 按键

按键包含：

- 显示/隐藏：只在剧情段落中出现，不在检查点中出现（段落和检查点定义可见v2-剧情模块）
- **跳过**：点击跳过按钮即可跳转至下一个检查点
- **回顾**：不在剧情段落中出现，不在检查点中出现

##### 2.2.3 剧情与操作逻辑

###### 2.2.3.1 字幕、音频与蓝牙信号

- 播放页面屏幕下方显示字幕，字幕文件与每个段落对应
- 音频：每个段落、检查点有对应的音频
  - 每个段落的音频与段落时长相等
  - 检查点为循环音频
- 蓝牙信号：特定段落有蓝牙信号，蓝牙信号与段落时长相等
- 字幕、语音、蓝牙信号均需要做时钟同步

###### 2.2.3.2 段落及检查点

- 段落：一个动画motion3配置为一个段落
  - 段落完成：用户首次播放完一个段落后，该段落为完成
- **检查点**：两个段落之间或者开头、结尾，检查点展示检查点选项页面
  - 用户进度：用户在当前剧情中已完成的最后一个**段落后**的检查点
  - 检查点区分普通检查点（非结束检查点）和结束检查点
    - **结束检查点**：一段剧情(story)中结尾最后的一个检查点
    - **普通检查点**：除了一段剧情(story)中结尾的检查点以外的检查点
  - 检查点效果：见figma和Live2d/语音交互效果说明
  - 检查点页面交互
    - 检查点页面播放idle动画或者指定图片、音频播放指定循环音频
    - 检查点页面的详细内容请参见Live2d/语音交互效果说明
    - 用户没有点击的情况下，按照默认设置播放对应的字幕（或对应无字幕，停留当前页面对应时间）
    - 在用户可点击的情况下（展示对应时间之后），用户点击即跳转下一字幕
    - **回顾**
      - 点击"回顾"可回顾本段落字幕
      - 回顾过程中暂停字幕播放（保持当前字幕不变，计时暂停）
      - 停止回顾，则回到检查点页面及计时状态，从断点时间继续播放字幕

###### 2.2.3.3 用户进度和跳过

- 用户进度
  - 用户进度的粒度为段落，用户每完成一段剧情（到达剧情后的检查点），该剧情段落即为完成
  - 用户每次进入时，都从头开始剧情
- 跳过条件
  - 用户进度已完成需跳过的段落
  - 若用户进度未完成需跳过的段落，此时"跳过"按键变灰不可点击
- 跳过内容
  - 如果当前是剧情播放中，则跳转到段落后的检查点（条件：已完成播放段落）
  - 如果当前是检查点页面，则跳转到段落后的检查点（条件：已完成下一个段落）
- 用户是否完成，从服务端用户进度中读取

##### 2.2.4 蓝牙信号

在剧情播放时，有同步的蓝牙信号，蓝牙可通过开关控制。

###### 2.2.4.1 蓝牙配置文件

蓝牙信号为每一个剧情段落一个，时间与剧情段落一致，检查点没有蓝牙信号。蓝牙信号配置文件为bluetoothConfig.xlsx，每个sheet为一个剧情，每个sheet中用live2d motion文件对应蓝牙信号文件名。

bluetoothConfig.xlsx示例：

| motion文件 | 蓝牙文件 |
|---|---|
| ch01_v0_stage_sec01_m01 motion3.json | ch01_v0_stage_sec01_m01 bluetooth.json |
| ch01_v0_stage_sec01_m02 motion3.json | ch01_v0_stage_sec01_m02 bluetooth.json |
| ch01_v0_stage_sec01_m03 motion3.json | ch01_v0_stage_sec01_m03 bluetooth.json |
| ch01_v0_stage_sec01_m04 motion3.json | ch01_v0_stage_sec01_m04 bluetooth.json |

###### 2.2.4.2 控制信号文件

蓝牙信号文件为json格式，字段含义如下：

| key | 含义 | 备注 |
|---|---|---|
| MotionFile | 对应的live2d motion文件名 | 字符串 |
| MotionDuration | 剧情段落时长（单位秒） | double |
| IntervalMs | 蓝牙发信间隔（单位毫秒） | int |
| SignalDurationMs | 持续时间（毫秒，对应蓝牙协议的图中黄框） | int |
| SignalDelayMs | 延迟时间（毫秒，对应协议的图中绿框） | int |
| Signal | 控制信号 | 双层列表，里层依次为摇摆、震动强度（控制档位，对应协议的图中红框）。例如：[[50,20],[40,30],[60,70]]为第一个信号的摇摆是50，震动20，第二个信号摇摆40，震动30，第三个信号摇摆60，震动70。数字为十进制，每个值的范围是0-100 |

蓝牙协议（无校验）

- 定时长模式
  - 控制档位(2o)：前8bit控制摇摆(入体)，合法范围 00000000 - 01100100，后8bit控制震动(体外)，合法范围 00000000 - 01100100。合法范围 0x00 0x00 ~ 0x00 0x64 / 0x01 0x00 ~ 0x01 0x64 / 0x02 0x00 ~ 0x02 0x64 / ... / 0x64 0x00 ~ 0x64 0x64
  - 持续时间(2o)：合法范围 0x0000 - 0xEA60，单位毫秒ms；0xFFFF代表持续震动不停止直到下一个接收到下一个信号
  - 延迟时间(1o)：合法范围 0x00 - 0x64，单位毫秒ms

*Signal信号的数量\*Interval等于MotionDuration

### 3. 回忆页（溯回星图）

#### 3.1 页面说明

- 档案选择页
  - 已解锁角色可以点击进入详情页
  - 未解锁角色显示为解锁图片，不可进入详情页
- 档案详情页
  - 档案详情页包含文案（印象碎片）、音频（羁绊回想）、图片（记忆残影）三个标签
  - 音频标签页音频为**主页音频汇总**。点击语音item可播放对应语音，语音播放的同时也播放字幕，没有特殊说明的情况下语音和字幕同时播放、同时停止。

#### 3.2 页面解锁

每个角色详情页由用户完成角色所对应的两个剧情来解锁：

- 用户账号的角色的两个剧情完成（进度为剧情的最终检查点）时，对应详情页解锁
  - orven：剧情1-1,1-2
  - solan：剧情1-3,1-4
  - varyn：剧情1-5,1-6
- 详情页的解锁只与剧情完成相关，与权限无关（账号剧情权限失效/被回收不影响详情页解锁）

### 4. 蓝牙与音频冲突处理

- **控制器预设波形**：用户使用控制器预设波形（与用户操作来源是控制器还是共鸣控制栏无关：共鸣同步时，音声与蓝牙信号同步；预设波形时，音声与蓝牙信号独立）
  - 进入航线页（剧情选择页）打断控制器预设蓝牙
    - 进入航线页后返回，不恢复蓝牙信号
  - 进入剧情模块（从主页进入剧情主页）、进入回忆页，不影响控制器预设蓝牙
- **共鸣音声及同步信号**：用户播放音声，如果使用同步信号，则同步信号与音声同步播放或打断
  - 共鸣信号进入剧情模块（从主页进入剧情主页）即打断
