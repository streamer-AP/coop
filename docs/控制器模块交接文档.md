## 一、模块职责

控制器模块负责：

1. **蓝牙设备管理** — 扫描、连接、断开 OMAO 设备
2. **波形控制** — 选择预设/自定义波形，控制双电机（摇摆 + 震动）
3. **常用波形配置** — 12 个快捷槽位（3 页 × 4 格）
4. **波形编辑** — 自定义波形（最多 18 个，最长 32 秒）
5. **使用日志** — 记录操作并回传云端
6. **信号仲裁** — 多信号源优先级裁决（story > resonance > preset）

需求文档：`需求文档/v2-控制器及蓝牙需求260204.md`
架构文档：`docs/技术架构决策.md`

## 三、核心数据结构

### 3.1 统一波形模型

预设与自定义波形共用一个结构，通过 `isBuiltIn` 区分：

```dart
// lib/features/controller/domain/models/waveform.dart
class Waveform {
  int id;
  String name;
  int durationMs;        // 默认 8000（单页 8 秒）
  bool isBuiltIn;        // true = 官方预设, false = 用户自定义
  List<WaveformKeyframe> keyframes;
}

class WaveformKeyframe {
  int timeMs;            // 毫秒时间点
  int swingValue;        // 摇摆值 15-100
  int vibrationValue;    // 震动值 15-100
}
```

波形播放逻辑：两个关键帧之间线性插值，以 200ms 间隔采样发送。

### 3.2 常用波形槽位

```dart
// lib/features/controller/domain/models/favorite_slot.dart
class FavoriteSlot {
  int page;       // 0-2（3 个标签页）
  int index;      // 0-3（每页 4 格，左上→右上→左下→右下）
  int waveformId; // 关联的波形 ID
}
```

### 3.3 使用日志

```dart
// lib/features/controller/domain/models/usage_log.dart
class UsageLog {
  int id;
  DateTime startTime;
  String signalMode;      // 'swing' / 'vibration' / 'both'
  int waveformId;
  int intensityLevel;
  int durationMs;
  String? deviceModel;
  String? deviceSerial;
}
```

记录粒度：`[signalMode + waveformId + intensityLevel]` 三者任一变化即新增一条记录。

## 五、信号流与优先级

### 5.1 信号源优先级

```
story (index=2) > resonance (index=1) > preset (index=0)
```

仲裁规则（`BleSignalArbitrator`）：
- `submitSignal()`：source.index >= activeSource.index 时接受，否则拒绝
- `releaseSource()`：释放后重置为静默（swing=0, vibration=0）

### 5.2 完整信号流

```
┌──────────────────┐
│  控制器页面       │ → BleSignal(source: preset)
│  共鸣播放器       │ → BleSignal(source: resonance)
│  Unity 剧情      │ → BleSignal(source: story)
└────────┬─────────┘
         ↓
  BleSignalArbitrator (仲裁)
         ↓
  BleSignalSender (200ms 定时发送)
         ↓
  BleDeviceProtocol.encodeSignal()
         ↓
  BLE Characteristic Write
         ↓
  OMAO 硬件设备
```

### 5.3 冲突切换规则

| 当前状态 | 触发事件 | 结果 |
|---------|----------|------|
| 预设波形 | 进入航线页 | 打断，信号停止 |
| 预设波形 | 进入剧情主页 | 不影响 |
| 预设波形 | 开启同步共鸣强度 | 切换到同步共鸣 |
| 同步共鸣 | 进入剧情主页 | 打断，信号停止 |
| 同步共鸣 | 音频暂停 | 信号暂停 |
| 同步共鸣 | 切换预设波形强度 | 切换到预设波形 |
| 剧情信号 | 退出剧情 | 信号停止 |
| 任意 | 蓝牙断开 | 双电机归零 |
| 任意 | App 进后台 | 继续发送 |

## 九、关键业务规则备忘

1. **波形控制默认值生命周期** = App 运行期（非持久化）
   - App 启动后默认：第一页 + 第一个波形 + 双电机关闭
   - 使用过程中记住用户最后选择，切换页面再回来时恢复

2. **连接后电机默认关闭** — 任何时候刚连接蓝牙，双电机强度为 0

3. **自定义波形限制**
   - 最多 18 个
   - 名称 ≤ 5 汉字（10 字符）
   - 4 页 × 8 秒 = 32 秒上限
   - 强度范围 15-100 整数
   - 两点间线性插值，200ms 采样间隔

4. **常用波形补位** — 移除某格后，同页剩余波形向前自动补位

5. **日志粒度** — `[signalMode + waveformId + intensityLevel]` 三者任一变化 = 新记录

6. **设备绑定** — 不自动重连，每次启动需用户手动连接
