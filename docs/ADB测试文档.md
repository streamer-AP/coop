## 一、测试环境准备

### 1.1 ADB 基础命令

```bash
# 安装 APK
adb install -r build/app/outputs/flutter-apk/app-release.apk

# 启动 APP
adb shell am start -n com.example.omao_app/.MainActivity

# 截图
adb exec-out screencap -p > screenshot_$(date +%s).png

# 录屏
adb shell screenrecord /sdcard/test_record.mp4

# 推送测试文件到设备
adb push test/integration/test_assets/ /sdcard/Download/omao_test/

# 查看日志
adb logcat -s flutter | tee test_log.txt

# 清除应用数据（重置测试环境）
adb shell pm clear com.example.omao_app
```

### 1.2 测试文件准备

将 `test/integration/test_assets/` 中的所有文件推送到设备：

```bash
adb push test/integration/test_assets/ /sdcard/Download/omao_test/
```

测试文件清单见 [附录A](#附录a-测试文件清单)。

### 1.3 前置条件

- Android 真机，API Level >= 24
- 蓝牙 4.0+ (BLE)
- 已授权存储/蓝牙/通知权限
- OMAO 蓝牙设备（控制器/剧情蓝牙相关测试）

## 附录B: ADB 批量测试脚本

```bash
#!/bin/bash
PACKAGE="com.example.omao_app"
TEST_DIR="/sdcard/Download/omao_test"
SCREENSHOT_DIR="./screenshots"

mkdir -p $SCREENSHOT_DIR

case "$1" in
    push) adb push test/integration/test_assets/ $TEST_DIR/ ;;
    screenshot) adb exec-out screencap -p > "$SCREENSHOT_DIR/${2}_$(date +%Y%m%d_%H%M%S).png" ;;
    reset) adb shell pm clear $PACKAGE ;;
    start) adb shell am start -n $PACKAGE/.MainActivity ;;
    stop) adb shell am force-stop $PACKAGE ;;
    *) echo "用法: $0 {push|screenshot <name>|reset|start|stop}" ;;
esac
```
